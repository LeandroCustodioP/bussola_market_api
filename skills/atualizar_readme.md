# Habilidade: Atualizar README.md

Sempre que uma nova funcionalidade ou um marco importante for concluído, você DEVE atualizar o arquivo `README.md` na raiz do projeto. Siga este padrão de organização:

1. **Seção de Status:** Mantenha um resumo rápido do que o projeto faz atualmente.
2. **Seção de Como Instalar/Rodar:** Instruções claras para configurar o ambiente (venv, requirements).
3. **Seção de Estrutura do Projeto:** Um breve resumo da hierarquia `src-layout`.
4. **Seção de Roadmap/Work Items:** Liste o que já foi entregue e o que está planejado para as próximas etapas.

**Importante:** Nunca apague informações anteriores; apenas acrescente as novas seções, garantindo que o documento permaneça como uma fonte de verdade para qualquer pessoa que chegue ao repositório.
